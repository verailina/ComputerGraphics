Предисловие
Задача на представление и преобразование цвета, цветовые пространства, гамма-коррекцию, использование формата YUY2, а также визуализацию без задержки реакции элементов пользовательского интерфейса.

Условия
Элементы пользовательского интерфейса приложения
1. Окно приложения с поддержкой изменения размеров
2. Масштабируемая область, в центре которой рисуется изображение после обработки
4. Прокрутка изображения при малых размерах окна
5. Элемент либо несколько для выбора обрабатываемого файла из списка исходных
6. Переключатель между коэффициентами стандарта BT.601 и BT.709
7. Слайдер для настройки гаммы и текущее значение гаммы (в виде подписи)
8. Изменение параметров приводит к перерисовке изображения, обеспечивающей мгновенный отклик на последующие действия пользователя

Логика приложения
Ваше приложение должно визуализировать результаты преобразования входных файлов из формата YUV (яркость и цветоразностные компоненты) в формат RGB (красный-зеленый-синий). В контексте обработки цифровых видеосигналов термин YUV соответствует «Y’CbCr». При этом Y' - компонента яркости с учетом гамма-коррекции, Cb и Cr - цветоразностные компоненты (синяя и красная, соответственно). В качестве входных данных предлагаются файлы формата PNG, в которых цвет представлен в виде четырех 8-битных компонент (четвертой компонентой является альфа-канал). Представленные данные предлагается интерпретировать как формат YUY2, т.е. интерпретировать группу каналов формата ARGB в виде YUVY. Предлагаются обработать следующий набор из 8 файлов (скачать архивом):

              

Перед преобразовании изображения из формата YUV в RGB необходимо распаковать YUV из формата 4:2:2 в формат 4:4:4 (убрать субдискретизацию), т.е. на каждый макропиксель YUYV получить 2 пикселя (Y1, U1, V1), (Y2, U2, V2). При этом можно использовать только исходные данные макропикселя {Y0, U0, Y1, V0}, но рекомендуется применять формулы усреднения, что дает лучшее качество. Например, для первого пикселя брать U и V макропикселя, а для второго - полусумму значений текущего и соседнего справа макропикселя. Таким образом, промежуточное изображение в формате YUV 4:4:4 будет иметь в 2 раза большее по ширине разрешение и 3 канала на пиксель. Из этого изображения, путем преобразования цвета из пространства YUV в RGB, необходимо получить полноцветное изображение. Для этого необходимо применить преобразование цветового пространства


В указанном преобразовании используются два коэффициента, определяемые разными стандартами. Предлагается использовать наиболее распространенные: ITU-R BT.601 телевидения стандартной чёткости (Kb = 0.114, Kr = 0.299) и ITU-R BT.709 телевидения высокой четкости (Kb = 0.0722, Kr = 0.2126). Указанные коэффициенты определяют положение RGB куба в пространстве YUV.


Как правило, информация о яркости для дискретного Y кодируется в диапазоне значений от 16 до 235. Эта особенность принята в стандарте MPEG2 и появилась из-за специфики функционирования первых аналоговых устройств отображения. В стандартных электронно-лучевых телевизорах и мониторах значениями до 16 и более 235 кодируется дополнительная информация (сигналы строчной и вертикальной синхронизации, и др.) Кроме этого, для цветоразностных компонент используется диапазон значений от 16 до 240. Таким образом, при распаковке дискретного сигнала (в том числе исходных данных из представленных выше файлов) необходимо нормировать значения до указанных диапазонов. Итоговые компоненты, используемые в формулах, заданы в диапазоне: [0; 1] для Y и [-0.5; 0.5] для UV.

Большинство мониторов и ЖК-панелей имеют нелинейную гамму. Это связано с техническими особенностями используемых систем и физическими особенностями человеческого глаза (глаз наиболее чувствителен к изменениям яркостной составляющей при малой интенсивности). В результате, использование гаммы позволяет компенсировать недостаточную точность 8-битного линейного диапазона. Наиболее широко применяемым значением является 2.2 При этом под гамма-коррекцией подразумевается коррекция яркости цифрового изображения - возведение в обратную степень линейной яркости. В итоге, после преобразований (гамма-коррекция + гамма монитора) воспроизводимое изображение соответствует изображению с линейной яркостью.

Наглядным примером, для проверки соответствия гаммы монитора значению 2.2, является приведенное ниже изображение. Представленное изображение состоит из трех блоков RGB-цветов (слева направо): блок серого цвета (128, 128, 128), блок чередующихся белых и черных полос (0 и 255), блок цвета (186, 186, 186)  При некотором отдалении глаз перестает различать отдельные полосы средней части изображения и воспринимает усредненный по этой области цвет, т.е. воспринимается усредняя яркость. Если монитор имеет нелинейную гамму (2.2), то воспринимаемый цвет будет близок к правой части изображения. Если гамма монитора линейна, то цвет будет ближе к левой части изображения. Кроме этого, любой монитор предоставляет возможность регулировки гаммы (и калибровки). Большинство систем предполагает использование монитора с гаммой 2.2


Изображения, предлагаемые в задании для преобразования в RGB, созданы из изображений с линейной яркостью путем применения гамма-коррекции с произвольной гаммой в диапазоне от 1.0 до 4.0 (необходимо определить значение). Кроме этого, использовались различные стандарты преобразования цветовых пространств - BT.601 либо BT.709. Разработанное вами приложение позволяет вам проанализировать предлагаемые изображения и попробовать экспериментально определить исходные параметры преобразований. Правильность заключения о параметрах не оценивается. Оценивается правильность выполнения описанных в задании шагов и функциональность приложения. Кроме этого, при выполнении задания следует учитывать, что гамма монитора 2.2. То есть необходимо преобразовывать входную яркость в линейный диапазон (возводить в степень x, где x - неизвестная гамма) и, перед преобразованием в RGB, применять гамму-коррекцию (возводить значение яркости в степень 1 / 2.2).
Оценивать результат восстановления исходного RGB изображения предлагается визуально. Наиболее удачные, на ваше усмотрение, параметры (для каждого файла отдельно) требуется записать в текстовый файл FIT0201SURNAME_TVSet.txt и отправить вместе с проектом. Как указано ранее, для каждого входного файла указывается BT601 либо BT709 и значение гаммы, с которым было закодировано изображение. В качестве подсказки для определения параметров можно обратиться к нижеследующим изображениям, а также, посмотреть короткий ролик о создании фильма «Alice in Wonderland».

     

В нижеследующем ролике представлены результаты применения компьютерных технологий в современном кинематографе. Из этого ролика взяты некоторые закодированные изображения.


Дополнительная информация
Допускается уточнение условий задачи по электронной почте и на семинаре.
Обеспечение «отзывчивости» интерфейса подразумевает, что расчет изображения должен производиться так, чтобы не мешать мгновенной реакции элементов GUI на действия пользователя.
В архиве с проектом должен располагаться файл FIT0201SURNAME_TVSet.txt, где приведены параметры, полученные вами в ходе экспериментов над входными данными.
