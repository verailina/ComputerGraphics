Задача Морфинг (FIT0201SURNAME_Morph)
Срок выполнения

Крайний срок: 20:00 26 апреля (пятница)

Предисловие
Задача на двухмерный морфинг, дробно-линейные преобразования и алгоритмы фильтрации изображений.

Условия
Элементы пользовательского интерфейса приложения
1. В области рисования располагается изображение, вид которого определяется 6 точками (A, B, C, D, E, F), и не изменяется при масштабировании окна
2. По-умолчанию точки A, B, D, E расположены так, что задают квадрат размером 256 на 256 пикселей, а отрезок CF делит квадрат пополам
3. Точки задают форму изображения. При этом изменяется положение только двух (центральные точки: C и F)
4. Два слайдера задают положение точек C и F на горизонтальной прямой.
5. Две группы переключателей для выбора типа фильтрации изображения: layer (point/linear), mip (none, point, linear)

Логика приложения
Приложение визуализирует два смежных четырехугольника, вид которых задается пользователем. Закраска производится с использованием дробно-линейного преобразования, задающего текстурные координаты. Для определения цвета пикселя используется текстура и выбранный пользователем алгоритм фильтрации (смешивания значений выборок из текстуры). Далее приводятся примеры трех различных конфигураций расположения точек A, B, C, D, E, F
Необходимо вычислять дробно-линейное преобразование, переводящее четырехугольники ABCF, FCDE в четырехугольники [0.0;0.0]x[1.0;0.5], [0.0;0.5]x[1.0;1.0], соответственно. Преобразование определяется следующими выражениями:
Для текстурирования используется изображение условно большого разрешения (1024x1024) - изображение планеты Земля. Если текстурные координаты выходят за диапазон [0; 1], используется "зеркальный" тайлинг (значение текстурной координаты 0.4 эквивалентно 1.6, 2.4, 3.6 и далее по аналогии)

При размещении исходного изображения согласно текстурным координатам получается, что каждому пикселю соответствует группа текселей. Алгоритм фильтрации определяет способ формирования цвета по ней.

Для расчета среднего значения по группе текселей применяются различные алгоритмы. В задаче предлагается использовать метод построения мип уровней и использовать различные методы вычисления итогового цвета: с использованием и без мип-уровней, различные выборки в конкретном мип-уровне (ближайший сосед, билинейная фильтрация). Далее представлен пример мип-уровней одного изображения. Уровни нумируются от нуля, нулевой уровень - изображение максимального разрешения.

Каждый последущий мип-уровень строится из следующих соображений: размеры в 2 раза меньше по ширине и высоте, цвет текселя вычисляется по блоку 2x2. Допустимо брать четверть суммы значений пикселей блока, цветовые каналы при этом обрабатываются независимо. Мип-уровни необходимо строить на этапе загрузки исходного изображения, либо допустимо заранее рассчитать все мип-уровни и использовать готовые изображения (например, загрузить из файла) во время работы программы. На этапе визуализации, в зависимости от алгоритма фильтрации, делаются выборки значений из различных мип уровней. 

Рассмотрим назначение переключателей для выбора алгоритма фильтрации. Переключатель из группы layer задает тип фильтрации слоя (ближайший сосед или билинейная), а переключатель из mip - способ выбора слоев (нулевой или ближайший, или линейная интерполяция между ближайшими). Таким образом, в случае layer[linear] плюс mip[linear] необходимо использовать трилинейную фильтрацию. В этом случае, на этапе определения цвета обрабатываемого пикселя выбирается два соседних мип-уровня так, чтобы отношение размеров пиксель-тексель было как можно ближе к 1:1. Таким образом, текстурные координаты конкретного пикселя могут быть представлены тройкой (u, v, d), где d - значение level of detail, определяющее уровни для текстурных выборок - два мип-уровня (либо один мип уровень, в случае старшего или младшего уровня). После этого, при чтении значений из уровней применяется билинейная интерполяция, а полученные значения слоев поканально смешиваются линейной интерполяцией (относительно вещественной части d). Для определения d можно использовать максимум длины стороны пикселя в текстурном пространстве либо абсолютный максимум частных производных u и v по dx и dy в экранных координатах.

Пример использования мип-уровней представлен на следующем изображении - нижняя плоскость визуализирована с использованием трилинейной фильтрации.

Дополнительная информация
Допускается уточнение условий задачи по электронной почте и на семинаре.

Рекомендуется выполнить список необязательных подзаданий:
1. Слайдер для масштабирования квадрата A, B, E, F
2. Просмотр текстурных координат (закраска в зависимости от значений u и v)
3. Отображение цветом ближайшего мип-уровня для пикселя
4. Загрузка изображений пользователя
5. New Возможность выбора минимального мип-уровня (например, 4x4 - достаточно слайдера от 0 до K, где K - номер наименьшего мип-уровня)
6. Поддержка альфа-канала и отображение на цветном фоне
